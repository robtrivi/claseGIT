¿Qué es el Control de Versiones?

El control de versiones es un sistema que registra los cambios realizados sobre un archivo o conjunto de archivos a lo largo del tiempo, de modo que puedas recuperar versiones específicas más adelante.

  • Mantiene un historial completo de cambios.
  • Permite revertir archivos a estados anteriores.
  • Facilita el trabajo colaborativo.

Importancia en el Desarrollo de Software

En proyectos de software, múltiples desarrolladores trabajan en el mismo código fuente. El control de versiones ayuda a coordinar el trabajo y a evitar conflictos.

  • Mejora la colaboración entre equipos.
  • Permite experimentar sin riesgo.
  • Facilita la identificación de errores.

Problemas sin Control de Versiones

Sin un sistema de control de versiones, el desarrollo puede volverse caótico y propenso a errores.

  • Sobrescritura accidental de archivos.
  • Dificultad para rastrear cambios y su autoría.
  • Falta de respaldo en caso de errores.

¿Qué es Git?

Git es un sistema de control de versiones distribuido, gratuito y de código abierto, diseñado para manejar todo tipo de proyectos con velocidad y eficiencia.

  • Creado por Linus Torvalds en 2005.
  • Permite que cada desarrollador tenga una copia completa del historial del proyecto.

Características Clave de Git

  • Distribuido: Cada copia es un repositorio completo.
  • Rápido: Operaciones locales y eficientes.
  • Seguro: Historial inmutable y verificación de integridad.

Cómo Funciona Git

Git maneja los datos como un conjunto de instantáneas (snapshots) del proyecto a lo largo del tiempo.

  • Repositorio Local: Tu copia en tu máquina.
  • Repositorio Remoto: Copia compartida en un servidor.
  • Snapshots: Estado del proyecto en un momento dado.

Git se puede instalar en múltiples sistemas operativos. A continuación, se muestran los pasos básicos para cada uno.

Windows

  • Descarga el instalador desde git-scm.com/download/win.
  • Ejecuta el instalador y sigue las instrucciones.

macOS

  • Usa Homebrew: `brew install git`.
  • O descarga el paquete desde git-scm.com/download/mac.

Linux

  • Debian/Ubuntu: `sudo apt-get install git`.
  • Fedora: `sudo dnf install git`.
  • Arch: `sudo pacman -S git`.

Configuración Inicial

Después de la instalación, configura tu nombre de usuario y correo electrónico: 

git config --global user.name "Tu Nombre"
git config --global user.email "tuemail@example.com"

Iniciando un Repositorio

Para iniciar un nuevo repositorio en un directorio existente: 

git init

Seguimiento de Archivos

Agrega archivos al área de preparación (staging area): 

git add archivo.txt

Verifica el estado de los archivos: 

git status

Realizando Commits

Guarda los cambios en el repositorio: 

git commit -m "Mensaje descriptivo"

Visualizando el Historial

Muestra el historial de commits: 

git log

Concepto de Ramas

Las ramas permiten desarrollar funcionalidades aisladas unas de otras.

  • Facilita el trabajo en paralelo.
  • Permite experimentar sin afectar la rama principal.

Comandos de Ramas

Crear una nueva rama: 

git branch nueva-rama

Cambiar a una rama existente: 

git checkout nueva-rama

Fusión de Ramas

Fusiona los cambios de una rama en la rama actual: 

git merge nombre-de-la-rama

Si hay conflictos, Git te indicará cuáles archivos necesitan ser resueltos manualmente.

GitHub es una plataforma de alojamiento de código que permite a los desarrolladores colaborar en proyectos utilizando Git.

  • Ofrece repositorios remotos para compartir código.
  • Incluye herramientas para gestión de proyectos.
  • Facilita la colaboración y revisión de código.

Git vs GitHub: Git es una herramienta de control de versiones, mientras que GitHub es un servicio que aloja repositorios Git y ofrece funcionalidades adicionales.

Creación de una Cuenta

Regístrate en github.com para crear una cuenta gratuita.

Configuración de Claves SSH

Las claves SSH permiten autenticarte sin necesidad de ingresar tu usuario y contraseña cada vez.

  1. Genera una clave SSH en tu máquina: 
    ssh-keygen -t ed25519 -C "tuemail@example.com"
  2. Copia la clave pública y agrégala a tu cuenta de GitHub.

Autenticación con GitHub

Después de configurar las claves SSH, puedes conectarte a GitHub de forma segura.

Conectando Repositorios

Para enlazar tu repositorio local con uno remoto en GitHub: 

git remote add origin git@github.com:usuario/repositorio.git

Subiendo Cambios

Para enviar tus commits al repositorio remoto: 

git push -u origin master

El flag `-u` establece `origin master` como la rama por defecto para `git push`.

Clonando Repositorios

Para obtener una copia de un repositorio existente: 

git clone git@github.com:usuario/repositorio.git

Actualizando el Repositorio Local

Para actualizar tu repositorio local con los cambios del remoto: 

git pull

Forks y Pull Requests

Los forks permiten copiar un repositorio para trabajar en él sin afectar al original. Los pull requests se usan para proponer cambios al repositorio original.

  1. Realiza un fork del repositorio.
  2. Clona tu fork y realiza cambios.
  3. Envía un pull request al repositorio original.

Gestión de Issues

Los issues son una forma de rastrear tareas, mejoras y errores para tu proyecto.

  • Reportar bugs.
  • Sugerir nuevas funcionalidades.
  • Discutir implementaciones.

Revisiones de Código

Las revisiones permiten mejorar la calidad del código mediante comentarios y sugerencias antes de fusionar cambios.

  • Comentarios línea por línea.
  • Aprobación o solicitud de cambios.
  • Discusión colaborativa.

Buenas Prácticas y Consejos

  • Mensajes de Commit Claros: Describe qué cambiaste y por qué.
  • Uso Efectivo de Ramas: Utiliza ramas para desarrollar nuevas características o corregir errores.
  • Resolución de Conflictos: Comunícate con tu equipo y resuelve conflictos lo antes posible.
  • Mantener el Repositorio Limpio: Elimina archivos innecesarios y utiliza `.gitignore` para excluir archivos temporales.
